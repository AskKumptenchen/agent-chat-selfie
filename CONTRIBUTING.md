# Contributing

Thank you for your interest in contributing to Chat Selfie.

Chat Selfie is an open-source project, and contributions of many kinds are welcome, including code, documentation, prompt design, integrations, adapters, workflow ideas, bug reports, and design discussions.

## Ways To Contribute

You can contribute by:

- opening an issue to report a bug or suggest an improvement
- submitting a pull request for a focused change
- improving documentation, onboarding, or examples
- refining persona, mood, heartbeat, or delivery workflows
- adding or improving image-generation integrations and adapters
- exploring new directions such as short video generation or lower-cost designs

## Before You Start

For small improvements, feel free to open a pull request directly.

For larger features, architectural changes, or workflow redesigns, opening an issue first is recommended so the direction can be discussed before implementation.

## Pull Request Guidelines

Please try to keep pull requests:

- focused on a single improvement or clearly related set of changes
- easy to review, with clear intent and minimal unrelated edits
- documented when behavior, contracts, prompts, or workflows change

When opening a pull request, include:

- a short summary of what changed
- why the change is helpful
- any important usage notes, limitations, or follow-up ideas

## Issue Guidelines

When opening an issue, it helps to include:

- what problem you are seeing
- what behavior you expected instead
- relevant screenshots, examples, or reproduction details when available
- any proposed direction if you already have one in mind

## Areas Especially Welcome

Contributions are especially welcome in these areas:

- documentation clarity and first-time onboarding
- long-term persona and emotional continuity
- mood modeling and emotional state persistence
- prompt refinement and output consistency
- heartbeat and proactive companionship flows
- compatibility with built-in, remote, and local image-generation routes
- cost efficiency, performance, and extensibility

## Community Expectations

Please be respectful, constructive, and collaborative in issues and pull requests.

Good-faith discussion, careful review, and clear communication are valued more than perfect first drafts.
